## 1. 项目概述

### 1.1 背景与目标
针对校园内二手交易信任度低、信息碎片化的问题，本项目旨在构建一个基于 SpringBoot 3 和 Vue 3 的轻量化交易平台。核心目标是提供一个**代码流畅、逻辑闭环、操作便捷**的校园内部市场。

### 1.2 核心技术栈
- **前端**: Vue 3 (Composition API) + Vite + Element Plus + Pinia + Vue Router
- **后端**: Spring Boot 3 +  Mybatis-Plus + JWT
- **数据库**: MySQL 8.0

---

## 2. 业务流程与角色

### 2.1 核心业务流程 (交易闭环)
1. **发布阶段**: 卖家发布商品  -> 商品上架展示。
2. **商议阶段**: 买家搜索/筛选 -> 查看详情 -> 发起咨询(私信) -> 达成意向。
3. **交易阶段**: 买家下单 -> 选择交易方式 -> 线下/线上对接 -> 确认收货。
4. **评价阶段**: 双方互评 -> 信用分自动更新。

### 2.2 用户角色
- **普通用户**: 注册、登录、买/卖商品、信用分维护。
- **系统管理员**: 平台监控、违规处理、统计看板。

## 3. 详细功能需求

### 3.1 用户管理模块 (User Management)
#### 3.1.1 注册与登录
- **输入**: 手机号/用户名、密码、确认密码。
- **逻辑**: 
  - 用户名唯一性校验。
  - 密码强校验（至少6位数字/字母）。
  - 基于 JWT 的无状态认证，Token 有效期 24 小时。
- **输出**: 注册成功并自动登录，返回 UserInfo 与 Token。

#### 3.1.2 个人信息管理
- **功能**: 
  - 头像上传（限制 4MB，格式 jpg/png）。
  - 昵称、联系方式（微信/QQ/手机号）修改。
  - 校区绑定（下拉选择）。

#### 3.1.3 信用评级体系
- **基础分**: 初始 80 分。
- **奖励规则**: 每完成一笔订单 +2 分，收到好评 +1 分。
- **惩罚规则**: 恶意取消订单（订单确认后取消）-10 分，被举报核实违规 -20 分。
- **展示**: 在商品详情页展示“信用极好90-100/优秀80-89/良好70-79/一般60-69/糟糕0-59”。

### 3.2 商品管理模块 (Item Management)
#### 3.2.1 商品发布
- **表单项**: 标题（20字内）、详情描述、分类（数码/生活/文体等）、价格（Decimal）、原价（参考）、新旧程度（下拉：全新/九成新等）、图片（最多5张）、交易区域。
- **规则**: 价格必须 > 0，至少包含一张图片。

#### 3.2.2 商品管理与状态机
- **状态流转**: 
  - `ACTIVE` (在售): 发布成功后的初始状态。
  - `PENDING` (待支付): 买家点击“支付”后，商品暂时锁定。
  - `SOLD` (已售出): 交易确认完成。
  - `OFFLINE` (已下架): 卖家手动撤回。

### 3.3 搜索与筛选模块 (Search & Filter)
- **全局搜索**: 顶部搜索框，支持标题与描述的模糊匹配（MySQL `LIKE %content%` 或全文索引）。
- **分类筛选**: Element Plus 侧边栏分类选择。
- **排序规则**: 支持“最新发布”、“价格最低”、“价格最高”、“信用最高”排序。

### 3.4 交易管理模块 (Transaction Management)
#### 3.4.1 商品咨询
- **形式**: 简易即时消息系统。
- **功能**: 
  - 留言板模式或私信模式。
  - 支持发送文本。

#### 3.4.2 交易方式与流程
- **方式**: 
  - 校园面交（推荐）：买卖双方约定地点。
  - 自提：买家到卖家处自提
  - 送货上门：填写校区、寝室号。
- **确认收货**: 买家在“我的订单”中点击确认，系统自动将商品状态改为 `SOLD` 并增加双方信用分。

### 3.5 通知系统 (Notification System)
- **类型**: 
  - **系统消息**: 欢迎语、规则更新。
  - **交易消息**: “您发布的商品有新咨询”、“买家已确认收货”。
- **展示**: 导航栏右上角红点提醒，消息列表按时间倒序。

### 3.6 管理后台 (Admin Backend)
- **仪表盘**: 
  - 统计卡片：总用户数、今日新增商品、交易完成总额。
- **管理功能**: 
  - **用户审计**: 封禁/解封违规用户。
  - **内容管理**: 强制下架虚假、违规商品。
  - **数据表单**: Element Plus 表单支持分页、关键字搜索、日期筛选。

## 4. 数据实体设计 (Schema Abstract)

### 4.1 User (用户)
`id, username, password, avatar, contact, campus, credit_score, status, create_time`

### 4.2 Item (商品)
`id, seller_id, title, description, category_id, price, original_price, condition_level, status, view_count, create_time`

### 4.3 Order (订单)
`id, item_id, buyer_id, seller_id, transaction_type, status, amount, create_time, update_time`

---

## 5. 非功能性需求

### 5.1 性能与稳定性
- **前端优化**: 使用 Vite 预构建，图片懒加载，减少首屏加载时间。
- **后端优化**: 使用 Spring Boot 缓存（可选）优化分类数据读取，数据库索引优化。

### 5.2 易用性 (UX Guidelines)
- **设计风格**: 清新校园风，主色调建议为活力绿或学术蓝。
- **响应式**: 必须兼容手机端浏览器（由于是轻量化平台，移动端访问频率更高）。
- **Element Plus**: 统一使用组件库的反馈机制（Message, Notification, Loading）。

---

## 6. 验收标准

1. **核心路径流畅**: 从注册到发布商品再到模拟买家完成交易，整个过程无 Bug 阻塞。
2. **代码规范**: 后端 Controller-Service-Repository 分层清晰；前端目录结构规范。
3. **数据准确**: 信用分计算逻辑与数据库记录完全一致。
4. **管理能力**: 管理后台能准确执行封号和下架操作。
